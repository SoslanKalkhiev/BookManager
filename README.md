# 📚 Менеджер любимых книг

**Техническое задание (README.md)**

## 1. Общая информация

**Название проекта:** Менеджер любимых книг
**Тип приложения:** Веб-приложение
**Технология:** ASP.NET MVC

### Описание

Приложение предназначено для учета прочитанных пользователем книг, их структурирования и организации в пользовательские подборки. Пользователь может самостоятельно добавлять книги, редактировать информацию о них, а также оставлять заметки.

⚠️ Хранение полного текста книг в рамках проекта не предусмотрено.

---

## 2. Цели и задачи проекта

### Цель:

Создать удобное веб-приложение для ведения личной библиотеки книг.

### Основные задачи:

* Учет прочитанных книг
* Классификация книг по пользовательским подборкам
* Добавление и редактирование информации о книгах
* Хранение пользовательских заметок
* Реализация системы пользователей и разграничения доступа

---

## 3. Функциональные требования

### 3.1 Страницы приложения

1. **Главная страница**

   * Отображение списка книг пользователя
   * Поиск по книгам (название, автор)
   * Быстрый доступ к подборкам

2. **Список объектов (книг)**

   * Отображение всех книг пользователя
   * Фильтрация и сортировка

3. **Страница книги**

   * Подробная информация:

     * Название
     * Автор
     * Жанр
     * Год
     * Описание
   * Пользовательские заметки
   * Принадлежность к подборкам

4. **Страница редактирования книги**

   * Изменение данных книги
   * Редактирование заметок

5. **Страница добавления книги**

   * Форма добавления новой книги

6. **Страница подборок**

   * Создание и управление подборками
   * Добавление книг в подборки

7. **Страница "О разработчике"**

   * Информация о разработчике проекта

---

### 3.2 Регистрация и авторизация

* Регистрация пользователя:

  * Логин
  * Email
  * Пароль
* Авторизация
* Хранение пользователей:

  * База данных или JSON

---

### 3.3 Разграничение доступа

* Каждый пользователь имеет доступ только к своим данным:

  * Книги
  * Подборки
  * Заметки

---

### 3.4 Работа с данными

* Добавление книг
* Редактирование книг
* Удаление книг
* Добавление заметок
* Работа с подборками

---

## 4. Хранение данных

Возможны два варианта:

### Вариант 1: База данных

* SQL Server / SQLite
* Таблицы:

  * Users
  * Books
  * Collections
  * Notes

### Вариант 2: JSON

* Хранение данных в файлах
* Используется для упрощенной версии проекта
